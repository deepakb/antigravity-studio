# SKILL: Design Token Architecture

## Overview
Enterprise **Design Token** system built on **style-dictionary** by Amazon. Implements three-layer token architecture (Primitive → Semantic → Component), compiling to CSS custom properties, TypeScript constants, and Tailwind v4 theme config from a single JSON source of truth.

## 1. Three-Layer Token Model

```
tokens/
├── primitive/           ← Raw values — NEVER used directly in code
│   ├── colors.json      → blue.500, gray.900, violet.600...
│   ├── spacing.json     → 1, 2, 4, 8, 12, 16, 24...
│   └── typography.json  → font-size.sm, line-height.normal...
│
├── semantic/            ← Intent-based — what developers reference
│   ├── colors.json      → color.brand.primary → {value: "{blue.600}"}
│   ├── spacing.json     → space.page-gutter → {value: "{spacing.6}"}
│   └── typography.json  → text.body → {value: "{font-size.base}"}
│
└── component/           ← Component-scoped — optional fine-grained control
    ├── button.json      → button.bg.default → {value: "{color.brand.primary}"}
    └── card.json        → card.padding → {value: "{space.6}"}
```

## 2. style-dictionary Configuration

```javascript
// style-dictionary.config.mjs
import StyleDictionary from "style-dictionary";

export default {
  source: ["tokens/**/*.json"],
  platforms: {
    // CSS Custom Properties (consumed by Tailwind v4 + components)
    css: {
      transformGroup: "css",
      prefix: "ds",
      files: [{
        destination: "src/design-system/tokens/tokens.css",
        format: "css/variables",
        options: { outputReferences: true }
      }]
    },
    // TypeScript ESM (consumed by JS code)
    typescript: {
      transformGroup: "js",
      files: [{
        destination: "src/design-system/tokens/tokens.ts",
        format: "javascript/esm"
      }]
    },
    // Tailwind v4 theme extension
    tailwind: {
      transformGroup: "js",
      files: [{
        destination: "src/design-system/tokens/tailwind-tokens.js",
        format: "javascript/module"
      }]
    }
  }
};
```

## 5. Tailwind v4 Integration

```css
/* src/app/globals.css */
@import "tailwindcss";
@import "../design-system/tokens/tokens.css";

@theme {
  /* Map compiled tokens to Tailwind theme */
  --color-brand-primary: var(--ds-color-brand-primary);
  --color-surface-base:  var(--ds-color-surface-base);
  --color-text-primary:  var(--ds-color-text-primary);
  --color-feedback-error: var(--ds-color-feedback-error);

  /* Spacing from tokens */
  --spacing-page-gutter: var(--ds-space-page-gutter);
}
```

## 6. Dark Mode Token Switching

```css
/* tokens.css — generated by style-dictionary */
:root {
  --ds-color-brand-primary: oklch(0.62 0.21 290);
  --ds-color-surface-base:  oklch(0.985 0 0);
  --ds-color-text-primary:  oklch(0.14 0 0);
}

[data-theme="dark"] {
  --ds-color-brand-primary: oklch(0.75 0.18 290); /* Lighter for dark bg */
  --ds-color-surface-base:  oklch(0.14 0 0);
  --ds-color-text-primary:  oklch(0.96 0 0);
}
```

## 7. Token Usage Rules
- ✅ `bg-[--ds-color-brand-primary]` — reference CSS var via Tailwind arbitrary
- ✅ `className={styles.card}` with `background: var(--ds-color-surface-raised)`
- ❌ `bg-violet-600` — never use primitive token name directly in components
- ❌ `style={{ background: "#7c3aed" }}` — never hardcode hex values
